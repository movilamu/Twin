## 📌 Overview

**Twin** builds a personalized **household digital twin** for disaster resilience. Instead of generic checklists like *"store 3 days of water,"* it generates a live, grounded, AI-reasoned resilience profile specific to **your household** — your floor level, your dependents, your medical equipment, and your region's real hazard data.

### The Problem

Disaster-preparedness guidance is almost universally **generic** — designed for a city or region, not a household. People with mobility needs, medical equipment, young children, or elderly family members receive the exact same advice as everyone else, regardless of whether it's actually relevant to them.

**Twin closes that gap** by grounding every recommendation in the household's real details and the region's real hazard data — then making it actionable through a one-tap emergency panel, available at the moment it matters most.

## ✨ Key Features

| Feature | What It Does |
|---|---|
| 🧮 **Household Resilience Score** | A 0–100 score across flood, seismic, and power-outage risk — generated by an LLM reasoning over your household details *and* real regional hazard data, not a static formula |
| 🌊 **"What If" Simulation** | Ask a free-form question (*"what if the power goes out for 3 days?"*) and get a streamed, numbered, **time- and color-coded** action plan |
| 🆘 **Panic Button** | A persistent floating emergency panel available on every page |
| 📞 One-tap click-to-call | Instantly reach local emergency numbers |
| 🏥 Nearest hospital finder | Free OpenStreetMap/Overpass data, severity-based search radius, live map |
| 💊 Age-aware medicine guidance | Covers common conditions like fever and cold — deliberately excludes numeric dosages; always defers exact dosing to a pharmacist, doctor, or product label |
| 📡 Offline fallback | Cached hospital results and static emergency numbers work even without a network connection |
| 🔒 **Private by Design** | Supabase magic-link auth with Row Level Security — each household's data is scoped to `auth.uid()`, never world-readable |
| 🔗 **Shareable Score Card** | Share your resilience score via the Web Share API or clipboard |
| 👋 **Guided Onboarding** | First-visit walkthrough plus a privacy / decision-support disclaimer, shown once per browser |

## 🏗️ Architecture

```
┌──────────────┐      ┌───────────────────┐       ┌──────────────────┐
│   Browser    │ ───▶ |  Next.js Route   |  ───▶ │   Groq (LLM)     │
│  (React UI)  │      |  Handlers (API)   │       │  Structured JSON │
└──────────────┘      └───────────────────┘       │  + Streaming     │
       │                       │                  └──────────────────┘
       │                       ▼
       │               ┌───────────────────┐
       │               │ Supabase Postgres │
       │               │  (RLS per user)   │
       │               └───────────────────┘
       │
       ▼
┌───────────────────────┐
│ OpenStreetMap         │
│ Overpass API (free)   │
└───────────────────────┘
```

### Data Flow, End to End

1. **Auth** — User signs in via Supabase magic link, then fills out the household setup form (Zod-validated on both client and server)
2. **Persist** — `POST /api/household` saves the profile, scoped to the user's `auth.uid()`
3. **Score** — `POST /api/resilience-score` builds a prompt combining household data with static regional hazard data, calls Groq in **structured JSON mode**, Zod-validates the response (one retry on malformed output), then persists and returns it
4. **Simulate** — `POST /api/simulate` streams a numbered, time-coded narrative via Server-Sent Events, rendered live as tokens arrive
5. **Panic Panel** — Queries the free Overpass API directly from the client, using the browser's geolocation (requested only on demand, never passively)

## 🚀 Quick Start

### Prerequisites

- Node.js 18+
- A free [Groq](https://console.groq.com/) account and API key
- A free [Supabase](https://supabase.com/) project

### 1. Clone & Install

```bash
git clone https://github.com/<your-username>/<your-repo>.git
cd <your-repo>
npm install
```

### 2. Configure Environment Variables

```bash
cp .env.example .env.local
```

Fill in the keys described in [Environment Variables](#-environment-variables) below.

### 3. Set Up the Database

Run `supabase/schema.sql` in your Supabase project's SQL Editor — see [Database Setup](#-database-setup) for details.

### 4. Run Locally

```bash
npm run dev
```

Open **[http://localhost:3000](http://localhost:3000)** — it redirects to `/landing`.

## 🔑 Environment Variables

| Variable | Required | Description |
|---|:---:|---|
| `GROQ_API_KEY` | ✅ | API key from [console.groq.com](https://console.groq.com/keys). Server-side only — never exposed to the client. |
| `NEXT_PUBLIC_SUPABASE_URL` | ✅ | Your Supabase project URL (Project Settings → API) |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | ✅ | Your Supabase anon/public key — safe for client exposure; access is enforced by Row Level Security, not key secrecy |

> ⚠️ **Never commit `.env.local`** — it's already covered by `.gitignore`.

## 🗄️ Database Setup

1. Open your Supabase project → **SQL Editor** → New query
2. Paste and run the contents of [`supabase/schema.sql`](./supabase/schema.sql)
3. Verify with `supabase/reset-and-verify.sql` if you need to confirm policies applied cleanly (safe to re-run)
4. In **Authentication → Providers**, confirm Email is enabled (default) for magic-link sign-in
5. In **Authentication → URL Configuration**, add local and production URLs to **Site URL** and **Redirect URLs** (e.g. `http://localhost:3000/**` and `https://<your-app>.vercel.app/**`)

**Tables created:**

| Table | Purpose |
|---|---|
| `households` | One row per saved household profile, scoped to `auth.uid()` |
| `resilience_scores` | One row per generated score, scoped to `auth.uid()` |

Both tables enforce **Row Level Security** — a user can only read or write their own rows.

## ☁️ Deployment (Vercel)

1. Push this repository to GitHub
2. Import the repo in [Vercel](https://vercel.com/) — framework auto-detects as Next.js
3. Add the three environment variables above under **Project Settings → Environment Variables** (check both **Production** and **Preview**)
4. Update Supabase **Redirect URLs** to include your Vercel deployment URL
5. Deploy 🎉

`vercel.json` sets a 10-second max duration on the AI route handlers to stay within Vercel's Hobby plan limits.

## 🔐 Security & Privacy

- **Row Level Security everywhere** — household and score data is private per-user via `auth.uid() = user_id`, enforced at the database layer, not just client-side gating
- **On-demand location only** — requested solely when the hospital finder is opened, never tracked passively
- **Responsible medical guidance** — medicine guidance deliberately excludes numeric dosages; always defers to a pharmacist, doctor, or product label
- **Transparent disclosure** — a first-visit notice explains data usage and clarifies this is decision support, not a replacement for emergency services
- **Server-side secrets** — API keys never leave the server; the Supabase anon key is safe for client exposure because access is enforced by RLS, not key secrecy
